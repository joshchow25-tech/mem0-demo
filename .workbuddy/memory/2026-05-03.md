# 2026-05-03 工作日志

## 记忆层优化（agent.py + memory_manager.py）

1. **对话存档过滤**：`agent.py` 新增 `_should_archive()` 方法，闲聊（<5字、命中黑名单词、LLM回复<20字）不写 conv_archive，减少无意义向量写入。
2. **检索去重+rerank**：`memory_manager.py` `search_memory()` 合并 mem0 画像 + conv_archive 结果后，按文本去重（保留最高 score），再按 score 降序排列返回 top_k。
3. **API 返回 `memory_archived` 字段**：`chat()` 返回值增加布尔字段，前端/日志可直观感知是否触发存档。

## 检索质量优化（config.py + rag_knowledge_base.py + memory_manager.py）

4. **RAG 相关度阈值**：`config.py` 新增 `RAG_SCORE_THRESHOLD`（默认0.4）、`MEM0_SCORE_THRESHOLD`（默认0.3）；
   `rag_knowledge_base.py` `search()` 先过阈值再排序，日志打印过滤前后条数。
5. **记忆检索阈值 + conv_archive 降权**：`memory_manager.py` `search_memory()` 过滤 < 阈值结果；
   conv_archive score ×0.85 降权，优先展示提炼后的用户画像；
   `format_memory_context()` 按【用户画像记忆】/【历史对话摘要】分类展示，并标注相关度百分比。

## 性能优化（agent.py）

6. **去掉重复检索**：`search()` 结果通过 `format_context_from_results()` 复用，不再调两次。
7. **并行检索**：记忆 + RAG 用 `ThreadPoolExecutor` 同时跑（embedding 串行→并行）。
8. **性能埋点**：`chat()` 和 `stream_chat()` 每次请求在日志打出各阶段耗时（检索/构建Prompt/LLM1/LLM2/存档）。
